Programmatic access

Use Kura without the dashboard

Everything you do here you can do from a terminal, a script, or an AI agent — against the same API, with the same authorization, masking, and audit. This page shows the three machine surfaces and how to authenticate to them.

Step one

Get a token

Every surface authenticates with a short-lived bearer token. You issue your own by signing in through your identity provider:

kura login --server {{.ServerURL}}

This opens your browser for the OAuth/OIDC handoff, then caches the token locally (owner-readable only). The CLI attaches it to every request automatically — you never paste it anywhere. The token is short-lived; when it expires, run kura login again. The dashboard you are reading uses this very same cached token, server-side.

Surface one

The CLI

The kura binary is the primary interface. After kura login, orient yourself and read data — all output is Markdown by default, or --json for scripts:

kura status            # identity + landscape briefing
kura whoami            # the principal the server resolves you to
kura query patient     # a masked, bounded page of records
kura show patient p1   # one masked record
kura log --actor alex@client.example   # filter the audit log

Reads are masked to your roles and every access is audited, exactly as in this dashboard. See the Machine interface section of the documentation for every verb.

Surface two

The HTTP API

The CLI and this dashboard are both just clients of the JSON API kura serve exposes. Call it directly by sending your token as a bearer credential. The API base is:

{{.ServerURL}}/api/

Every request carries the token in the Authorization header:

curl -H "Authorization: Bearer <token>" \
  {{.ServerURL}}/api/whoami

curl -H "Authorization: Bearer <token>" \
  "{{.ServerURL}}/api/patient?limit=50"

curl -H "Authorization: Bearer <token>" \
  {{.ServerURL}}/api/patient/p1

The server masks every field and records every access on its side — there is no unmasked or unaudited path onto the data, whichever client you use.

Surface three

The MCP server coming soon

For AI agents, kura mcp will expose the same gated operations as Model Context Protocol tools, so an assistant can query and act on your store under the same authorization, masking, and audit as any other client — no separate, weaker path to the data.

kura mcp   # arrives in a later release (Phase 5)

It will project from the same core operations registry as the CLI and the API, so its tools stay behaviorally identical to the verbs above.

For agents

Self-describing commands

So an agent need not scrape --help, kura agent-context emits a versioned, machine-readable JSON description of every command, flag, and surface Kura exposes:

kura agent-context
